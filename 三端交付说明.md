# 红薯下载三端交付说明

## 交付范围

- Windows V2.4.0：便携桌面应用，默认系统下载目录，可自选目录。
- Android V1.1.0：原生 APK，默认 `Download/红薯下载`，可用系统选择器授权其他目录。
- iPhone V0.1.0：SwiftUI 源码预览，默认本机 Files 目录，可自选 Files/iCloud Drive 目录。
- 三端没有设备互联、云同步或账号体系，下载、设置和历史分别留在各自设备。

## 测试门槛

- 公共解析与存储单元测试全部通过。
- Windows 最终 EXE 必须启动、检查主题/设置完整性并做真实下载回归。
- Android 最终 release APK 必须在 Redmi K60 真机安装，测试主界面、设置、目录授权和真实下载。
- iPhone 必须通过 GitHub macOS/Xcode 的无签名模拟器编译；尚未使用 Apple 证书和 iPhone 真机验收，因此只发布源码预览，不声称可安装 IPA。

## 本次验收结果

- 自动测试：`pytest` 11 项全部通过，Python 源码编译检查通过。
- Windows：V2.4.0 最终 EXE 启动正常，主界面和设置页截图检查通过；真实下载 AirPods Pro 笔记成功，系统下载目录生成 5 张 PNG 与 `文案.txt`。
- Android：V1.1.0 release APK 编译和 lint 通过，在 Redmi K60 安装成功；系统目录授权成功，真实下载小米15 Ultra 笔记成功，授权目录生成 6 张 PNG 与 `文案.txt`。
- iPhone：macOS 15 / Xcode 16.4 的 iPhone Simulator 无签名编译通过；真机安装、视频播放和 iCloud Drive 持久写入仍保留为签名后的验收项。

## 数据与输出

每条笔记目录只包含原始媒体和 `文案.txt`。文案包含标题、正文、话题。文件夹名为 `评数字-赞数字-标题-作者`。历史 JSON 只集中保存下载网址、笔记 ID 和标题，不散落到各笔记目录。
