# xhs-dl V2.3 桌面版：设计与使用说明

## 一、设计思路

这个版本面向不想接触命令行的 Windows 用户。核心目标是“打开、粘贴、下载”三步完成，同时让保存位置、速度和更新检测保持可控。

视觉延续上一版的冷灰蓝、轻拟物和克制阴影语言，但从浏览器页面改为独立桌面窗口。主界面只展示高频动作，低频选项集中到右上角“设置”按钮。

## 二、功能与数据设计

- 本地桌面窗口：双击 `小红书无水印下载器.exe` 启动。
- 设置按钮：管理保存位置、下载速度和自动更新检测。
- 默认目录：当前 Windows 用户的 `Downloads` 文件夹。
- 逐条进度：显示当前完成数量和每条结果。
- 剪贴板识别：检测到小红书链接时显示“粘贴并采集”，也可以手动粘贴后点击“开始采集”。
- 原始媒体：使用独立安装的 XHS-Downloader 本地引擎。
- 文案文件：每篇笔记生成 `文案.txt`，包含标题、正文和话题。
- 文件夹命名：`评128-赞3560-帖子标题-作者`，缺少公开数据时使用“未知”。
- 集中历史：所有成功笔记统一写入 `%LOCALAPPDATA%\xhs-dl\history.json`，每条仅保存下载网址、笔记 ID 和标题。
- 下载目录整洁：笔记文件夹只放媒体和 `文案.txt`，不放 JSON；下载根目录也不生成批次 JSON。

## 三、首次使用

1. 解压便携版压缩包。
2. 如果电脑尚未安装本地引擎，先双击 `一键安装V2.bat`。
3. 双击 `小红书无水印下载器.exe`。
4. 粘贴一条或多条小红书分享内容。
5. 点击“开始下载”。

默认由软件按链接数量自动选择随机间隔：1 条直接采集，2–20 条使用 35–55 秒，21–50 条使用 55–85 秒，超过 50 条使用 110–160 秒。也可以在设置中手动指定档位。

## 四、修改保存位置

1. 点击窗口右上角“设置”。
2. 点击“选择文件夹”。
3. 选择目标目录并点击“保存设置”。

设置保存在 `%LOCALAPPDATA%\xhs-dl\settings.json`，下次启动自动沿用。

## 五、更新检测

软件默认在启动后后台读取 GitHub Releases 的最新版本号。发现新版本时，窗口右上角会出现版本提示，点击可打开正式发布页。检测失败不会影响下载，也不会自动安装未知文件。可以在设置中关闭检测。

## 六、输出示例

```text
Downloads/
└── 评128-赞3560-AirPods隐藏功能-多多数码屋/
    ├── 图片_1.png
    ├── 图片_2.png
    └── 文案.txt
```

集中数据位于：

```text
%LOCALAPPDATA%\xhs-dl\
├── settings.json
└── history.json
```

## 七、边界说明

软件只处理可公开访问的笔记，不绕过登录、私密、删除、地区或权限限制。无水印指直接保存公开页面提供的原始媒体，不会擦除作者主动嵌入图片的署名或 Logo。
