# 红薯下载使用手册

## 你能做什么

- 免登录下载公开小红书笔记的原始图片和视频
- 粘贴一条链接、多个链接或整段分享文本
- 使用命令行、浏览器可视化界面，或让 Codex/OpenClaw 调用共享技能
- 自动去重、慢速执行，并将结果清单保存到本地

## 第一次安装

1. 解压 GitHub Release 的 `红薯下载-v2.4.0-portable-windows.zip`。
2. 双击 `一键安装V2.bat`。
3. 安装完成后，双击 `红薯下载.exe`。

安装过程会创建独立 Python 3.12 引擎环境，不会替换系统 Python，也不需要小红书账号、密码或 Cookie。

## 桌面应用

双击 `红薯下载.exe`，粘贴分享文本后点击“开始采集”。右上角“设置”可以选择保存文件夹、调整速度、查看使用说明、手动检测更新或开关启动检测；“拟态悬浮 / 克制玻璃”按钮可以切换主题。默认保存到当前 Windows 用户的“下载”文件夹。

## Android 应用

1. 安装发布页的 `红薯下载-android-v1.1.0.apk`。如果系统提示未知来源安装，只为你信任的这个安装包临时授权。
2. 打开“红薯下载”，粘贴分享文本后点“开始采集”；也可以从其他应用把文字分享到本应用。
3. 新安装默认进入 `Download/红薯下载`。设置页既可修改 Download 下的子文件夹名称，也可点“选择其他下载目录”并在系统页面授权一个文件夹。
4. 顶部可以切换“拟态悬浮 / 克制玻璃”，主题会保留。页面可上下滚动，小屏也能访问所有操作。

Android 版不需要小红书登录，不申请读取全部相册或全部文件的权限，只通过系统下载存储写入自己的结果。

## iPhone 应用

苹果源码位于 `ios/`。默认下载到“文件”App 中可见的本机 `红薯下载` 目录；在设置中点“选择文件夹”可授权“我的 iPhone”或 iCloud Drive 里的其他目录。授权、设置和历史只保存在本机，不与电脑或安卓同步。

iOS V0.1.0 已使用 GitHub 的 macOS/Xcode 构建机做模拟器编译检查；真机安装包必须使用你自己的 Apple 开发者签名生成，不能在 Windows 上伪造可安装 IPA。

## 旧版可视化界面

双击 `启动Web版.bat`，浏览器会打开 `http://127.0.0.1:5678`。粘贴分享文本后直接点“开始下载”即可，文件默认进入当前 Windows 用户的“下载”文件夹。保存位置和速度收在高级设置中；右上角可以切换“拟态悬浮”和“克制玻璃”。下载时右侧进度条会持续显示当前状态，即使任务很快完成也会保留清楚的完成过渡。

## 普通粘贴模式

双击 `启动无水印版.bat`，粘贴分享文本后按回车。默认保存到启动目录下的 `xhs_downloads`。

## 输出结果

每篇笔记文件夹只包含原始媒体和 `文案.txt`，不再放 JSON。文件夹使用 `评128-赞3560-标题-作者` 格式。成功历史统一保存在 `%LOCALAPPDATA%\xhs-dl\history.json`，字段为下载网址、笔记 ID 和标题。

## 无水印是怎么实现的

它不是识别水印后再涂抹图片，而是由本地引擎解析公开分享链接，从笔记的公开媒体信息中选取未叠加平台账号水印的原始媒体地址，再直接保存原文件。这样不会产生修补痕迹；作者主动做进图片的署名、Logo 和品牌字样不会被删除。

## AI 与小龙虾调用

- 共享技能目录：`skills\xhs-download`
- Codex 调用：`$xhs-download`
- OpenClaw 全局安装：`openclaw skills install <项目路径>\skills\xhs-download --global`

当前电脑尚未安装 OpenClaw，并且 Node 版本低于技能部署要求，因此这里只交付可迁移技能，没有伪造“已接通”状态。

## 常见问题

- 公开链接失效：重新从小红书生成分享链接。
- 部分内容失败：等待后用 `slow` 模式重试。
- 创作者自己的标志还在：那是原图内容，不属于平台账号水印。
- 想使用在线解析网站：默认不使用，因为链接会发给第三方；只有你明确同意后才考虑。
