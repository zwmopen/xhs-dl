## 二、技术原理（为什么去水印网站能做到）

### 2.1 水印从哪来

小红书的图片水印**不是前端加的，是 CDN 服务端烧录的**。但关键在于：**CDN 上同时存了有水印和无水印两个版本**，区别在 URL。

### 2.2 两个版本的 URL 区别

| 版本 | URL 特征 | 来源 | 有无水印 |
|------|---------|------|---------|
| Web SSR 版 | `.../notes_pre_post/...!h5_1080jpg` | `__INITIAL_STATE__.noteData.data.noteData.imageList[].url` | **有水印** |
| 原图版 | `.../notes_pre_post/...`（无后缀或不同 scene） | API 返回的 `imageList[].info_list[]` 里特定 scene 的 url | **无水印** |

V1 拿到的是第一版（带 `!h5_1080jpg`），所以有水印。去水印网站拿到的是第二版。

### 2.3 去水印网站的技术原理

去水印网站不需要登录账号，它们做的是：

1. **调用小红书 Web API**（而不是解析 SSR HTML）
   - 核心接口: `https://edith.xiaohongshu.com/api/sns/web/v1/feed`（获取笔记详情）
   - 这个 API 返回的 JSON 里，每张图片有 `info_list` 字段
   - `info_list` 包含多个 scene 的 URL，其中 `WB_DFT`、`WB_PRV`、`CRD_PRV` 等 scene 是无水印原图

2. **实现请求签名算法**
   - 小红书 Web API 需要两个签名请求头: `X-S`（签名）、`X-T`（时间戳）
   - 签名算法基于 `window._webmsxyw` 函数（小红书前端 JS 里）
   - 去水印网站逆向了这个算法，可以离线计算签名
   - 签名输入: URL path + query 参数 + 请求体
   - **这是核心技术壁垒**：谁实现了签名算法，谁就能直接调 API 拿无水印图

3. **关键参数**
   - `xsec_token`: 短链接 302 重定向自带，或长链接 URL 里带（必须，否则 404）
   - `a1`: Cookie 里的反爬参数（部分方案需要，部分不需要）
   - `webId`: Cookie 里的设备标识（同上）
   - `X-S` / `X-T`: 请求签名（核心）

### 2.4 老方法（traceID，已失效）

2022年5-6月前，小红书网页 HTML 里能直接找到 traceID，用它可以直接拿无水印图。现在网页里已移除 traceID，这个方法失效了。当前主流是调 Web API + 签名。

### 2.5 为什么不需要登录

- `xsec_token` 是分享链接自带的，相当于一个临时访问令牌
- 签名算法 `X-S`/`X-T` 可以离线计算（不依赖登录态）
- 所以只要实现了签名算法 + 有 xsec_token，就能调 API 拿无水印原图
- 这就是为什么去水印网站不需要登录那么多账号

## 六、关键技术细节（给实现者的参考）

### 6.1 小红书 Web API 签名算法要点

如果选择自己实现签名（方式B），需要关注:

- 签名函数: `window._webmsxyw`（小红书前端 JS）
- 签名输出: `X-S`（base64编码的签名）、`X-T`（毫秒时间戳）
- 签名输入: 请求的 URL path + query
- 实现方式:
  1. 用 PyExecJS 执行原始 JS（最简单但依赖 Node.js）
  2. 用 Python 重写签名算法（复杂但无外部依赖）
  3. 直接用 XHS-Downloader 已实现的版本（参考其 source/ 目录）

### 6.2 无水印图片 URL 特征

- 有水印: `https://sns-webpic-qc.xhscdn.com/.../notes_pre_post/...!h5_1080jpg`
- 无水印: `https://sns-webpic-qc.xhscdn.com/.../notes_pre_post/...`（去掉 `!h5_1080jpg`）—— **但这招可能已失效，因为水印是烧录的不是后缀加的**
- 正确做法: 调 API 拿 `info_list` 里 `WB_DFT` scene 的 URL

## 七、注意事项

1. **GPL-3.0 协议**: XHS-Downloader 是 GPL-3.0，如果复用其代码，xhs-dl 也必须 GPL-3.0 开源。xhs-dl 当前是 MIT，需改 License 或只调用不复用代码。
2. **风控**: 实测时不要一次性测太多，每条间隔 10 秒以上。被风控后换 IP 或等几小时。
3. **链接时效**: 分享链接带日期信息，旧链接可能被风控。实测用最新生成的链接。
4. **Python 版本**: XHS-Downloader 要 3.12，用户环境是 3.10.11。这是第一个要解决的坑。
5. **用户偏好**: 用户要求"挨个去测，自己自检、检测，成功了再说，不成功就继续自己卷"。不要测一个就报喜，要测完一批给对比结果。

## 八、已完成的调研结论（省去重复搜索）

1. **去水印网站不需要登录账号**，因为它们实现了小红书 Web API 的签名算法（X-S/X-T），可以离线计算签名，配合分享链接自带的 xsec_token 调 API 拿无水印原图。
2. **核心区别**在 `info_list`：V1 取的是 `imageList[].url`（有水印），去水印版取的是 `imageList[].info_list[]` 里特定 scene 的 url（无水印）。
3. **traceID 已失效**（2022年后），现在主流是 Web API + 签名。
4. **XHS-Downloader 是当前最成熟的开源方案**，2.2+ 无需 Cookie，支持 API 模式，建议首选集成。
5. **tapsave.net 是推荐的在线工具**（CSDN 2026-07 实测），可作为备选。
6. 市面上有几十个去水印网站/API，但本质原理都一样（调 Web API + 签名），差别只在稳定性和速度。
