# xhs-dl 项目交接文档

> 交接日期: 2026-07-18 | 当前版本: V1.0.0 | 状态: 可用，已完成基础功能

---

## 一、项目概述

**xhs-dl** 是一个小红书笔记下载器，支持从 xhslink 短链接或分享文本中提取笔记正文和图片，保存到本地。无需登录账号。

- **CLI 版**: 命令行工具 `xhs-dl`
- **Web 版**: 浏览器界面 `xhs-dl-web`，粘贴链接点下载
- **Python SDK**: 可被其他项目 import 调用

### 已知限制（重要）

1. **图片带水印**: 小红书服务端在存储图片时就烧录了水印（半透明"小红书"字样），技术上无法通过修改 URL 去除。这不是代码 bug，是平台行为。
2. **视频笔记**: V1 仅下载封面图，不下载视频文件。
3. **风控**: 频繁请求会被临时拦截（返回安全页面/404），工具已内置 2-5 秒随机延迟。

## 二、技术原理

```
用户输入 xhslink.com 短链接
    ↓ requests.get 跟踪 302 重定向
完整 URL（含 xsec_token、app_platform 等全部参数）
    ↓ 用同一 session（模拟 iPhone Safari UA）GET 该 URL
HTML 页面（服务端 SSR 渲染，约 150KB）
    ↓ 正则提取 window.__INITIAL_STATE__ = {...}
JSON 数据（括号计数法确保完整匹配）
    ↓ 从 JSON 中提取笔记数据
路径1: state.noteData.data.noteData（新版，最常见）
路径2: state.note.noteDetailMap[noteId].note（旧版）
路径3: state.noteData.normalNotePreloadData（预加载，可能不完整）
    ↓ 提取图片 URL（imageList / infoList）
图片 URL 特征: https://sns-webpic-qc.xhscdn.com/.../notes_pre_post/...!h5_1080jpg
    ↓ HTTP→HTTPS + 带 Referer 头下载
本地文件
```

**关键发现**:
- 必须用短链接重定向后的**完整 URL**（带 xsec_token），直接用 note_id 访问 /explore/ 会被安全系统拦截到 404 页面
- 图片 URL 末尾的 `!h5_1080jpg` 是 CDN 认证后缀，**不能去掉**，去掉会 403
- 图片 URL 从 HTTP 换成 HTTPS 即可正常下载
- User-Agent 用 iPhone Safari 效果最好

## 四、核心代码说明

### 4.1 downloader.py（核心）

**XhsDownloader 类**:

```python
dl = XhsDownloader(
    output_dir="./xhs_downloads",  # 保存目录
    delay=(2, 5),                   # 请求间随机延迟
    on_progress=callback_fn,       # 进度回调
)

# 方式1: 传入链接列表
result = dl.download(["http://xhslink.com/o/xxx", ...])

# 方式2: 传入分享文本（自动提取链接）
result = dl.download_text("快存下！vivo X300 http://xhslink.com/o/xxx")
```

**关键方法**:
| 方法 | 作用 |
|------|------|
| `_resolve_url(url)` | 跟踪短链接 302 重定向，返回完整 URL |
| `_parse_note_id(url)` | 从 URL 正则提取 24 位 hex 的 note_id |
| `_extract_ssr_state(html)` | 括号计数法从 HTML 提取 `__INITIAL_STATE__` JSON |
| `_extract_note(state)` | 从 JSON 中按 3 条路径查找笔记数据 |
| `_get_media_urls(note)` | 从 imageList 提取图片 URL，兼容 infoList/info_list |
| `_download_file(url, path)` | 带 Referer 头下载，验证文件 >200B |
| `_sanitize(name)` | 清理文件名非法字符 |

**SSR 数据结构（实测）**:
```
window.__INITIAL_STATE__ = {
  "noteData": {
    "data": {
      "noteData": {           // <-- 主要数据在这里
        "title": "...",
        "desc": "...",
        "type": "normal",     // "normal" | "video"
        "user": {"nickname": "..."},
        "imageList": [
          {
            "url": "http://sns-webpic-qc.xhscdn.com/.../!h5_1080jpg",
            "url_default": "",   // SSR 中通常为空
            "infoList": [],       // SSR 中通常为空
            "fileId": "...",
            "width": 1080,
            "height": 1440,
          }
        ],
        "video": {...}        // 视频笔记才有
      }
    },
    "normalNotePreloadData": {  // 预加载（备用）
      "title": "...",
      "desc": "...",
      "imagesList": [{"url": "...", "width": ..., "height": ...}]
    }
  }
}
```

### 4.2 cli.py

argparse 定义，支持:
- 位置参数 `urls`: 链接或分享文本
- `-f/--file`: 从文件读取
- `-o/--output`: 保存目录
- 自动从文本中正则提取 URL

### 4.3 web/app.py

- 纯 Python 标准库 `http.server`，无额外依赖
- 内嵌 HTML/CSS/JS（小红书主题色 #ff2442）
- `GET /` → 返回页面
- `POST /api/download` → 接收 `{text, output_dir}`，同步下载后返回 JSON
- 端口 5678，启动后自动打开浏览器

## 六、待改进 / V2 方向

1. **视频下载**: SSR 中的 video 字段包含视频信息，但视频 URL 可能需要额外处理（consumer.originVideoKey 等字段）
2. **去水印研究**: 当前 CDN 返回的图片确实带水印。可能的探索方向:
   - 检查是否有其他图片尺寸/格式后缀（如原始图）能绕过水印
   - 研究 App 端 API 是否返回无水印图
3. **并发下载**: 当前是串行下载，可以加线程池
4. **进度显示**: Web 版目前是同步阻塞的，大量链接时浏览器会卡住等待，应该改成异步（SSE 或 WebSocket 推送进度）
5. **GUI 版本**: 用 PyQt/Tkinter 做桌面应用
6. **错误重试**: 加入自动重试机制
7. **Cookie/登录**: 可选支持登录后获取更多数据（如私密笔记）
8. **Docker 化**: 方便部署
9. **图片格式检测**: 根据 URL 后缀或 Content-Type 保存为正确格式（jpg/png/webp）

## 七、使用方法（给下一个人的快速上手）

```bash
# 安装
cd xhs-dl
pip install -r requirements.txt

# CLI
python -m xhs_dl.cli "http://xhslink.com/o/xxxxx"
python -m xhs_dl.cli -f links.txt -o ./output

# Web
python -m xhs_dl.web.app
# 打开 http://127.0.0.1:5678

# Python 调用
from xhs_dl import XhsDownloader
dl = XhsDownloader(output_dir="./notes")
result = dl.download(["http://xhslink.com/o/xxxxx"])
```
